# Documentation Standards

Reference for the `documentation-generator` skill. Covers Markdown formatting, doc types, file organization, and writing style for DZIRE_v1 project documentation.

## Doc Types and Locations

| Doc Type | Location | Naming |
|----------|---------|--------|
| README | `docs/README.md` | Fixed |
| Architecture guide | `docs/architecture.md` | Fixed |
| Changelog | `docs/changelog.md` | Fixed |
| Setup guide | `docs/setup.md` | Fixed |
| API reference | `docs/api-reference.md` | Fixed |
| Module docs | `docs/[module].md` | Per module |
| Skill docs | `skills/[skill-name]/SKILL.md` | Per skill |
| Workflow docs | `workflows/[workflow-name].md` | Per workflow |

## Markdown Formatting Rules

- H1 (`#`) — Document title only (one per file)
- H2 (`##`) — Major sections
- H3 (`###`) — Subsections
- Use fenced code blocks with language tags: ` ```python`, ` ```bash`, ` ```sql`
- Tables for structured data (3+ items with multiple properties)
- Bullet lists for 2+ unordered items
- Numbered lists for sequential steps

## README Structure

```markdown
# [Project Name]

> One-line description.

## Overview
[What this is and why it exists]

## Tech Stack
[Key technologies]

## Quick Start
[Install + run commands]

## Project Structure
[Key folders]

## Documentation
[Links to other docs]

## Contributing
[How to contribute]
```

## Changelog Format

```markdown
## [YYYY-MM-DD] — [Short description]

### Added
- [What was added]

### Changed
- [What was changed]

### Fixed
- [What was fixed]

### Removed
- [What was removed]
```

```markdown

## Overview
[System purpose and scope]

[Key components and their roles]

## Data Flow
[How data moves through the system]

## Dependencies
[External services and libraries]

## Decisions
[Key architectural decisions and rationale]
```

## Writing Style

- Use **present tense**: "The service handles..." not "The service will handle..."
- Use **active voice**: "The agent generates..." not "Documentation is generated by..."
- Be **concrete**: include actual file paths, commands, and examples
- Avoid vague language: "somehow", "various", "some cases"
- Keep sentences short (< 25 words preferred)
- Code references in backticks: `config/stack.config.json`

## File Path References

Always use relative paths from the document's location:
- From `docs/` → `../skills/skill-creator/SKILL.md`
- From `skills/[name]/` → `../../docs/architecture.md`
