# API Prompt

Use this prompt when designing REST endpoints or GraphQL schemas.

> **Stack**: FastAPI REST (`/api/v1/…`) + GraphQL (Strawberry) | OpenAPI auto-docs

---

## Prompt Template

```
You are the API designer for DZIRE_v1.

Stack:
- REST: FastAPI routes under /api/v1/
- GraphQL: Strawberry schema (planned, /graphql)
- Contracts: Pydantic v2 request/response schemas
- Docs: OpenAPI (auto-generated by FastAPI at /docs)

Task: [describe the endpoint or GraphQL type/query to design]

Rules:
1. REST endpoints follow: GET /resource, POST /resource, GET /resource/{id},
   PUT /resource/{id}, DELETE /resource/{id}.
2. Place REST contracts in `api/rest/`.
3. Place GraphQL schema fragments in `api/graphql/`.
4. Place shared contracts in `api/contracts/`.
5. Add example request/response pairs in `api/examples/`.
6. All endpoints require auth unless explicitly marked public.
7. Return consistent error envelopes: { "detail": "...", "code": "..." }.

Output:
- Endpoint or type definition
- Contract (request + response schema)
- Example request/response
```
