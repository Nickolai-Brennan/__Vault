# Project Startup Guide

## Purpose
Use this guide when turning a new idea into a real project.

## Startup Flow
1. Capture the idea
2. Complete the idea intake form
3. Generate the project brief
4. Select the stack
5. Create the repo scaffold
6. Create task breakdown
7. Create first build milestone

## Required Starting Files
- idea-intake-form.md
- project-brief-template.md
- project-brief-example.md
- README.md
- docs/index.md
- tasks/master-task-list.md

## Recommended First Questions
- What is the project?
- Who is it for?
- What problem does it solve?
- What is the simplest usable version?
- How will it make money?
- How will traffic/users arrive?
- What data does it need?
- What tools or APIs are required?

## First Build Milestone
The first milestone should produce:
- Working repo
- Basic documentation
- Core folder structure
- Environment setup
- Initial task list
- MVP scope

## MVP Rules
- Build the smallest useful version first
- Avoid adding secondary features too early
- Keep one clear primary user
- Keep one clear main outcome
- Track every feature as a task
- Document every major decision

## Output
At the end of startup, the project should have:
- Project brief
- Repo scaffold
- Stack decision
- Workflow assignment
- Task list
- Launch direction
